## 1. 总览

`$AA47` 是 Bank $02 的一个**多用途数据表**，负责存储各关卡的**球员阵容**（player roster）和**阵型/控制数据**。通过零页变量 `$26`（游戏总进度计数器）来控制读取哪个区段。

核心子程序全部位于 `bank_02.asm` ROM offset $482F–$48C6 (CPU $882F–$88C6)。

---

## 2. AA47 数据布局

所有偏移相对于 `$AA47`（ROM offset `$4A57`）：

```
$AA47  ┌─────────────────────────────────────────┐
+$00   │ 阵容 A: 11 字节球员 ID                  │  $26 ∈ {0, $0C} 使用
+$0B   │ 控制字节 A → $2C                        │
+$0C   │ 阵容 B: 11 字节球员 ID                  │  $26 = 6 使用
+$17   │ 控制字节 B → $2C                        │
+$18   │ 阵容 C: 11 字节球员 ID                  │  $26 = $10 使用
+$23   │ 阵型数据: 10 字节 → $0408 (RAM)         │  仅 $26 = $10 加载
+$2D   │ 控制字节 C → $2C                        │
+$2E   │ AA75 表: 匹配控制值 ($26 索引 → $2A)    │  所有路径都读取
       │  (紧随其后, 共 ≥32 字节)                 │
       └─────────────────────────────────────────┘
```

### 2.1 原始 ROM 数据 (从 BZK disasm 捕获)

```
阵容 A (+$00~+$0A): 02 03 04 05 06 07 08 09 0A 01 0B
控制 A (+$0B):      00
阵容 B (+$0C~+$16): 0F 0D 0E 14 10 0C 13 12 15 11 16
控制 B (+$17):      00
阵容 C (+$18~+$22): 22 1B 1C 14 1D 17 18 11 1A 01 15
阵型  (+$23~+$2C):  19 1F 10 12 13 16 1E 20 21 0F
控制 C (+$2D):      01
────────────────────────
AA75 表 (+$2E 起, $26=0..A):  00 00 00 00 00 00 01 01 01 01 01
```

> **注释**：CDL 标记此区域为 `-D1---`（data，有间接引用），与代码中 `LDA $AA47,X` 的绝对索引寻址一致。

## 3. $26 进度计数器 — 阵容路由逻辑

### 3.1 主入口 `$8855` (ROM $4865)

```
$8855: LDA $E4
       CMP $26
       BCS $88A8        ; 如果 $E4 >= $26 → 跳过阵容加载 (只设 $2A/$2B)
       
       LDA $26
       BEQ $887C        ; $26 == 0  → 用阵容 A (X=0)
       CMP #$06
       BEQ $8884        ; $26 == 6  → 用阵容 B (X=$0C)
       CMP #$0C
       BEQ $887C        ; $26 == $0C → 用阵容 A (X=0) ← 复用!
       CMP #$10
       BEQ $888C        ; $26 == $10 → 用阵容 C (X=$18) ← 含阵型加载
       JMP $A8A8        ; 其他 → 跳过
```

### 3.2 副入口 `$886E` (ROM $487E)

```
$886E: LDA $26
       CMP #$06
       BCC $887C        ; $26 < 6   → X=0    (阵容 A)
       CMP #$0C
       BCC $8884        ; $26 6..B  → X=$0C  (阵容 B)
       CMP #$10
       BCS $888C        ; $26 >= $10 → X=$18 (阵容 C)
       ; fall → $887C   ; $26 $0C..$0F → X=0 (阵容 A)
```

### 3.3 $26 路由总结

| $26 范围 | 入口点 | X 初值 | 读取源 | 阵型? |
|----------|--------|--------|--------|-------|
| 0 | 主/副 | $00 | 阵容 A (+$00) | 否 |
| 1–5 | 副 | $00 | 阵容 A (+$00) | 否 |
| 6 | 主/副 | $0C | 阵容 B (+$0C) | 否 |
| 7–B | 副 | $0C | 阵容 B (+$0C) | 否 |
| $0C–$0F | 主/副 | $00 | 阵容 A (+$00) | 否 |
| $10+ | 主/副 | $18 | 阵容 C (+$18) | ✅ 是 |

**关键发现**：
- 阵容 A 被两个不连续的 $26 区间复用 ($0 和 $0C)
- 阵容 C (`$26 >= $10`) 是唯一加载阵型数据的路径
- `$E4` 是一个"已加载标记"，如果 `$E4 >= $26` 说明当前阵容不需要重载

## 4. 子程序详细分析

### 4.1 `$88B7` — 加载 11 球员 ID 到 `$0300`

```
入口: X = 阵容在 AA47 中的起始偏移 (0/$0C/$18)
破坏: A, X, Y

$88B7: LDA #$0B          ; ED = 循环计数 (11 球员)
       STA $ED
       LDY #$00
.loop: LDA $AA47,X       ; 读球员 ID
       STA $0300,Y       ; 写 RAM 阵容区
       INX
       TYA
       CLC
       ADC #$0C          ; Y += 12 (每球员在 RAM 中占 $0C 字节)
       TAY
       CMP #$84
       BCC .loop         ; 循环直到 Y >= $84 (12×11=$84)
       RTS
```

**RAM 布局**:
```
$0300-$030B: 球员 0 (ID 在 $0300)
$030C-$0317: 球员 1
...
$0378-$0383: 球员 10
```

### 4.2 `$8893` — 加载阵型数据到 `$0408`

```
仅 $888C 路径 (X=$18 读取阵容 C 后 X=$23) 执行

$8893: LDY #$00
.loop: LDA $AA47,X       ; 读阵型字节
       STA $0408,Y       ; 写阵型 RAM
       INX
       TYA
       CLC
       ADC #$04          ; Y += 4 (每阵型条目 4 字节)
       TAY
       CMP #$28
       BCC .loop         ; 循环 10 次 (Y: 0→$24)
```

**RAM 布局**: `$0408-$042B` 10 条阵型记录，每条 4 字节。

### 4.3 `$88A3` — 读取控制字节

```
$88A3: LDA $AA47,X       ; 读取阵容后的控制字节
       STA $2C            ; → RAM $2C
       ; 接着 fall through 到 $88A8
```

### 4.4 `$88A8` — AA75 匹配控制

```
$88A8: LDX $26            ; X = 进度计数器
       LDA $AA75,X        ; $2A = AA75[$26]
       STA $2A
       LDA $26
       CLC
       ADC #$03
       STA $2B            ; $2B = $26 + 3
       RTS
```

**结论**: `$2A` = AA75 查表值 (推断为比赛半场控制: 0=上半场, 1=下半场等); `$2B` = `$26 + 3` (推断为关卡编号/难度)。

## 5. Bank $01 中的 $26 用法

`bank_01.asm` L22 (`$801E`):

```
$801E: LDA $0448
       LSR A
       LDA $26           ; 读取进度
       ROL A
       CLC
       LDX $0446
       CPX #$05
       ROL A
       STA $0660         ; 组合 → $0660 (球员状态/属性计算用)
```

**作用**: 将 `$26` 与 `$0448`(比赛状态?)、`$0446`(队伍索引?) 位运算合并成一个控制字节写入 `$0660`，用于球员属性/状态渲染。

## 6. $2A / $2B / $2C 含义推断

| RAM | 来源 | 值范围 | 推断含义 |
|-----|------|--------|----------|
| `$2A` | AA75[$26] | 0/1 | 比赛半场: 0=上半场, 1=下半场 |
| `$2B` | $26 + 3 | 3..$23+ | 关卡编号 (最小=3对应$26=0) |
| `$2C` | AA47 控制字节 | 0/1 等 | 阵容类型标记 (主/客/特殊?) |
