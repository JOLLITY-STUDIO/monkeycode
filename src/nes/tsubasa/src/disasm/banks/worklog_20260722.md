## #10 bank_02_nmi.ts 三个 NMI 核心函数空壳 → 完整实现

**现象**: 屏幕花屏，只有 fallback 测试图案（蓝色 tile 1-4）。

**根因**: bank_02_nmi.ts 的 NMI/PPU 渲染路径三个关键函数都是空壳，导致显示列表不填充 → PPU 无数据 → 花屏。

**修复详情**:

### `processDisplayList` ($800A, 约 80 行)

**之前**: 只有注释，缺少 tile 写入循环（`$8014-$8046`），直接跳到 `goto$804D`。显示列表完全没用。

**修复**: 完整实现 $8014-$8046 的外/内两层循环：
- 关闭 PPU 渲染 (`$2001=0`)
- 外循环遍历 display list 每组: 读 attr → bit7 决定 nametable → 写 PPUADDR
- 内循环逐 tile 写 PPUDATA（`attr & 0x3F` 个 tile）
- INX+3 跳过 attr+addr，检查下一组 attr 是否为 0 决定退出

### `nmiSubStateDispatch` ($8160)

**之前**: 空壳，无条件调用 `gotoNmiContinue(ctx)`。

**修复**: 完整实现 $8160-$81A7：
- `state & 0x80` → PPUADDR 直接写入模式：读 $79/$7A,X → 写 $2006，设 PPUCTRL=$20&$FC
- `state & 0x80 == 0` → 滚动模式：LSR $20 → 合并 scrollVal.bit0 → 设 PPUCTRL+PPUSCROLL

### `gotoNmiContinue` ($81A8)

**之前**: 空壳，逻辑不完整，无条件结束 NMI。

**修复**: 完整实现 $81A8-$81CA：
- `masked==0 || X==$13` → 结束 NMI，调用 `nmiBankSwitch(ctx, 0x18, 0)`
- 否则: timer+=3, `irqVal = masked<<1` → 写 $C000/$C001

**涉及文件**: `bank_02_nmi.ts`
**状态**: ✅ fixed

## #11 bank_00 sceneDataLoad $F5-$FF 控制码全部 stub → 完整实现

**现象**: 11 个控制码全部无实际逻辑，仅设置 `advance`，场景数据加载在碰到这些码时跳过关键处理。

**修复**: 从 ASM $87B8-$887F 逐条翻译：

| 码 | ASM 地址 | 功能 | advance |
|----|----------|------|---------|
| $F5 | $87B8 | 读取下一字节 → 设置显示控制 $4C (`byte==$FF?0:byte\|$80`) | 2 |
| $F6 | $87CB | clearPPUMode + saveContext(下一字节) 延迟 | 2 |
| $F7 | $87D9 | toggle $7B bit0，清 $7A/$44/$45 | 1 |
| $F8 | $87F8 | 读 2 字节→$EC/$ED，切 bank_02，调用 $A212 | 2 |
| $F9 | $8814 | 清 $5B bit2 + sceneFade(下一字节) | 2 |
| $FA | $881B | 置 $5B bit2 + sceneFade(下一字节) | 2 |
| $FB | $8831 | 读 $9085 ROM 数据初始化 PPU 脚本参数 | 1 |
| $FC | $8837 | clearPPUMode + delay 4，$51/$52 += $40，重设光标 | 1 |
| $FD | $8855 | sub88B1(清除显示区) + delay 4，重设光标 | 1 |
| $FE | $8862 | 读 2 字节 → 新 $4D/$4E (字节码流 JMP) | 0* |
| $FF | $8870 | clearTaskSlot + 终止字节码循环 | 0† |

> \* $FE 直接重设 dataPtr/dataOffset = 0  
> † $FF 设 dataOffset = 999 退出 while 循环

**涉及文件**: `bank_00_dispatch.ts`
**状态**: ✅ fixed
