# Worklog 2026-07-21 — 天使之翼 2 Bank 反汇编排错

## #1 场景状态机卡死在 `handleForward` (jumpIdx 僵直)

**现象**: 每帧日志 `dispatch: idx=1 → target=$818A`，`NO fn at $818B`，$27=1 不变化。

**根因**: `bank_00_dispatch.ts` `JUMP_TABLE` 6 入口 (idx=0~5)，`fns` 只注册了 `handleForward`(idx=0) 和 `sceneLoopEntry`(idx=5)。

**修复**: 从 ASM 补译 6 个函数：

| 入口 | 函数 | 作用 |
|------|------|------|
| $818B | `handleJump1` | $28/$29 比较 + SCENE_MODE_TABLE 查表 |
| $81AE | `handleJump2` | 直接设 jumpIdx=3 |
| $81B5 | `handleJump3` | mode=3 时设 idx=4 |
| $81DB | `handleJump4` | $28>$29 → advanceScene |
| $81E6 | `previousSceneTransition` | 切 bank_01 → $A015 → SCENE_NEXT_TABLE → sceneStateMachine |
| $8206 | `advanceScene` | 切 bank_01 → $A012 → SCENE_FLAG_C/D → INC $26 |

**涉及文件**: `bank_00_dispatch.ts`
**状态**: ✅ fixed

## #2 runner.ts 每帧重置 jumpIdx 导致状态机无法前进

**涉及文件**: `runner.ts`

## #3 $C4B9 跨 bank 调用未传递 X 寄存器 (bank 号)

**现象**: `previousSceneTransition` 中 `xcall('$C4B9')` 后 `xcall('$A015')` 路由错误：默认切到 bank_02 而非 ASM 指定的 bank_01。

**根因**: 6502 ASM `LDX #$01; JSR $C4B9` 将 MMC3 R7 切到 bank_01，但 TS 版 `xcall()` 未传 X 值。

**修复**: 修复以下调用点显式传 bank 号：

| 函数 | ASM X | 修复 |
|------|-------|------|
| sceneStateNormal ($80F5) | 2 | `xcall(ctx, rom, '$C4B9', 2)` |
| sceneStateNormal ($810A) | 2 | `xcall(ctx, rom, '$C4B9', 2)` |
| sceneStateNormal ($8117) | 1 | `xcall(ctx, rom, '$C4B9', 1)` |
| sceneStateNormal ($813D) | 1 | `xcall(ctx, rom, '$C4B9', 1)` |
| previousSceneTransition | 1 | `xcall(ctx, rom, '$C4B9', 1)` |
| advanceScene | 1 | `xcall(ctx, rom, '$C4B9', 1)` |
| setupTextDisplay ($8285) | 1 | `xcall(ctx, rom, '$C4B9', 1)` |
| saveBankAndWaitNmi ($9B07) | 6 | `xcall(ctx, rom, '$C4B9', 6)` |
| sceneTransitionSetup ($8920) | 6/saved | `xcall(ctx, rom, '$C4B9', 6)`/`savedBank` |
| waitNmi ($838A) | 2 | `xcall(ctx, rom, '$C4B9', 2)` |

**涉及文件**: `bank_00_dispatch.ts`
**状态**: ✅ fixed

## #9 sceneDataLoad const dataPtr + dispListEntry spin-wait 死循环

**现象**: 自动测试 F2 起每帧 `Assignment to constant variable`，scene $01 不推进。

**根因**:
1. `sceneDataLoad` 中 `const dataPtr` 被 `$F0` case 尝试 `+= 3` → TypeError
2. `dispListEntry` 的 do-while spin-wait 检查 `$0629 & 0x40`，TS 模拟器中无 PPU 消费 → 死循环

**修复**:
- `const dataPtr` → `let dataPtr`
- `$F0` case: `dataOffset = -1` → `dataOffset = 0`
- `dispListEntry`: 移除 spin-wait do-while

**涉及文件**: `bank_00_dispatch.ts`
**状态**: ✅ fixed
