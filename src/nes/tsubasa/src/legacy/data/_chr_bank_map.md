# 足球小将 II CHR Bank 分配 & 文字渲染机制

ROM: Captain Tsubasa II - Super Striker (Japan).nes
Mapper: MMC3 (4), CHR: 128KB (16 × 8KB = 64 × 2KB 子 bank), PRG: 256KB (16 × 16KB)

---

## MMC3 PPU 映射
| PPU 地址 | tiles | 区域 | MMC3 reg |
|---|---|---|---|
| $0000-$07FF | $00-$3F | BG 左半 | reg 0 |
| $0800-$0FFF | $40-$7F | BG 右半 | reg 1 |
| $1000-$17FF | $80-$BF | SPR 左半 | reg 2+3 |
| $1800-$1FFF | $C0-$FF | SPR 右半 | reg 4+5 |

## 场景 → Bank 分配推测
| 场景 | 需要什么 tile | 可能用的 bank |
|---|---|---|
| 标题画面 | logo、菜单字 | bank 0~1 |
| 球员状态画面 | 数字字体、UI 框、球员头像 | bank 0 (BG) + bank ? (SPR) |
| 比赛画面 | 球场、球员、球、特效 | bank 2~5 |
| 剧情过场 | 人物立绘、对白框 | bank 6+ |
| 不同关卡 | 不同球场草皮/天气 | 按关卡切换 |

## 数值渲染完整链路

### 总览

```
ROM 球员属性表 (marker $EC = 体力)
  → 16-bit 原值 $02EC (748)
  → CPU 数位分解 (百位=7, 十位=4, 个位=8)
  → 字节码编码 ($FF 模式: $FF $07 $04)
  → 解释器分发 ($86AF → $9A31)
  → MMC3 PRG bank 6 查元数据
  → 属性表 + 显示列表 → PPU → CHR tile 渲染
```

---

### 第零層：ROM 球員屬性表結構

多個 bank (03, 04, 05, 06) 存有球員數據，每個 bank 前端為指標表，指向各球員記錄。記錄內部用 **marker bytes** 編碼不同屬性：

| marker byte | 推測屬性 | 值範圍 | 範例 (ROM offset) |
|---|---|---|---|
| `$EC` | 體力 | `$0001`~`$FFFF` (16-bit) | bank 3 `$813C`: `EC 02` (=748) |
| `$ED` | ? | 1 byte | `$43`, `$27`, `$62` |
| `$E8` | ? (可能是記錄開頭) | 1 byte | `$01` |
| `$F3`, `$F4`, `$F5` | ? | 多 bytes | 多處 |
| `$F9`, `$FA`, `$FB` | ? | 多 bytes | 多處 |
| `$FC` | 記錄結束標記 | - | 各記錄末尾 |

**體力 `$EC` 分佈**：
- bank 3: `$8134-$814B` 等處
- bank 5: `$84A1` 等處
- bank 6: `$8062`, `$80A7`, `$8108` 等處

---

### 第一層：ROM 原始值 `$EC $02` → CPU 讀取

段位碼解釋器從 `$4D/$4E` 指向的字節流讀取 marker bytes 並識別屬性值。

---

### 第二層：數位分解 (16-bit → digits)

CPU 有專門子程序 (`bank_02` `$87BC`/`$87C3`):
```
LDX #$64    ; 100
LDY #$00    ; 計數器
```
748 = $02EC → 百位=7, 十位=4, 個位=8

---

### 第三層：字節碼編碼

分解後的 digits 通過三種模式寫入字節碼流:

| 模式 | 編碼 | 說明 |
|---|---|---|
| 單百位 | `$01`~`$7F` | digit 值直接當百位 |
| 單十位 | `$80`~`$FE` | (value & $7F) 當十位 |
| 雙位 | `$FF` + 2 bytes | `$FF` `百位` `十位` |

748 使用雙位模式: `$FF $07 $04`

---

### 第四層：字節碼解釋器 → PPU 渲染

```
$86AF (解釋器分發)
  │  讀 $FF → 雙位模式
  │  讀下兩 bytes: A=$07 (百位), X=$04 (十位)
  │  JSR $9A31
  ▼
$9A31 (兩位入口)
  │  STA $48     ; 百位=7
  │  STX $49     ; 十位=4
  │  JSR $9B07   ; 切換 PRG bank 6 (MMC3 reg7)
  │  JSR $9AB8   ; 讀百位 16 字節元數據 $B070→$062A
  │  JSR $9ADA   ; 讀十位 16 字節元數據 $B340→$062A+16
  │  JSR $C4B9   ; 恢復 PRG bank
  │  JSR $9A71   ; 處理 32 字節元數據緩衝
  ▼
$9A71 → $9AA2 (構建顯示列表)
  │  迴圈: Y=0..15 (百位), Y=16..31 (十位)
  │  每字節: AND #$0F → 低 nibble, ORA 屬性 → $05E8,Y
  │  查 $9EA2 屬性表決定 palette/flip/priority
  ▼
$9B28 / $9B5E (寫入 tile 索引)
  │  顯示列表結構 (每 3 bytes):
  │    $05E8,X = attr
  │    $05E9,X = Y 坐標
  │    $05EA,X = tile 索引 ($3x 範圍)
  ▼
NMI → PPU
  │  pattern table 0, tile $3A = 數字 "7"
  │  圖形: 7E 42 42 04 08 10 10 10
  │  視覺:
  │    ######
  │    #    #
  │    #    #
  │        #
  │       #
  │      #
  │      #
  │      #
```

### 數字元數據格式

PRG bank 6 中每個數字有 16 字節元數據，兩組（百位在 `$B0xx`，十位在 `$B3xx`）：

| 字節 | 百位 bank6+$1000 | 十位 bank6+$1300 |
|---|---|---|
| digit 0 | `0F 1A 18 30  0F 36 25 30  0F 21 31 30  0F 2B 10 30` | `0F 0F 0F 36  0F 0F 16 36  0F 0F 30 36  0F 0F 30 36` |
| digit 7 | `0F 0A 08 30  0F 36 25 30  0F 0C 01 30  0F 0C 0F 10` | `0F 0F 30 36  0F 0F 32 36  0F 0F 22 36  0F 0F 30 36` |

每組 4 字節對應一個片段行，`$0F` 開頭表示新片段起始，中間字節編碼 X 偏移和 sprite 屬性。

### MMC3 PRG Bank 切換詳解

```
$9B07: LDA $25    ; 保存當前 bank
       PHA
       LDX #$06   ; 選擇 PRG 8KB bank 6
       STX $25
       JSR $C4B9  ;

$C4B9: STX $25    ; 保存 bank 號到 $25
       LDA #$07   ; MMC3 寄存器 7 ($A000-$BFFF)
       ORA $22
       STA $8000  ; 選寄存器
       STX $8001  ; 設 bank 6 → $A000-$BFFF

結果: PRG bank 6 映射到 CPU $A000-$BFFF
      ROM 實際地址 = 16 + 6×0x2000 + 0x1000 = 0xD010
```

### 字節碼解釋器完整調度表

所有 `$E8`-`$FF` 的 opcode 通過 `bank_00.asm` 的跳轉表 (`$8537-$8544`) 分派：

| Opcode | 處理地址 | 功能 | 參數 |
|---|---|---|---|
| `$E8` | `$8575` | 腳本入口/控制 | 1 byte |
| `$E9` | `$8580` | ? | 1+ byte(s) |
| `$EA` | `$858D` | ? | ? |
| `$EB` | `$85C4` | ? | ? |
| `$EC` | `$85D2` | **播放音效** | `$01`-`$FE`: 音效 ID, `$FF`: 停止 |
| `$ED` | `$85EC` | **設置隊伍槽位** | 1 byte 隊伍 ID → `$0700,X` |
| `$EE` | `$8604` | ? | ? |
| `$EF` | `$8618` | ? | ? |
| `$F0` | `$862C` | 座標設置 | 2 bytes 讀取座標 |
| `$F1` | `$864A` | 初始化名字區域 | 讀 1 byte 參數 → 查表 |
| `$F2` | `$8678` | **設置文本行數** | 1 byte → `$55` |
| `$F3` | `$8682` | **數字顯示控制** | 見下表 |
| `$F4` | `$86B8` | **二級子跳轉** | 8 個子命令 (見下) |
| `$F5` | `$87B8` | 場景狀態 | 1 byte → `$4C` |
| `$F6` | `$87CB` | 等待 N 幀 | 1 byte 幀數 |
| `$F7` | `$87D9` | 動畫切換控制 | 切換 `$7B`/`$7A` |
| `$F8` | `$87F8` | **讀取 2 字節到 $EC/$ED** | 2 bytes → `$ED`/`$EC` RAM |
| `$F9` | `$8814` | **清除旗標 bit2** | `AND #$FB` → `$5B` |
| `$FA` | `$881B` | **設置旗標 bit2 + 加載場景** | `ORA #$04` + 1 byte 場景 ID |
| `$FB` | `$8831` | **遞迴腳本調用** | `JSR $9085` 嵌套執行 |
| `$FC` | `$8837` | **換行/滾屏** | 準備文本 + 等待 4 幀 + 光標下移 |
| `$FD` | `$8855` | 文本區域清除準備 | 調用 `$88B1` + 等待 4 幀 |
| `$FE` | `$8862` | **讀取跳轉地址** | 2 bytes → `$4E`/`$4D` (絕對跳轉) |
| `$FF` | `$8870` | **結束腳本** | 清 `$4D/$4E` → `JMP $9F7E` |

### `$F3` 數字顯示子模式

| 編碼 | 處理 | 說明 |
|---|---|---|
| `$F3 $00` | `JSR $9A35` | 處理現有緩衝（不載新值） |
| `$F3 $01`-`$7F` | `JSR $9A4C` | 單百位顯示 |
| `$F3 $80`-`$FE` | `JSR $9A60` | 單十位顯示 (值 = byte & `$7F`) |
| `$F3 $FF $hh $tt` | `JSR $9A31` | **雙位顯示**: A=百位(hh), X=十位(tt) |

### `$F4` 二級子跳轉表 (`$86C6`)

| 子索引 | 目標 | 功能 |
|---|---|---|
| 0 | `$86D5` | 調用 `$B020`（在當前 bank） |
| 1 | `$86DD` | `JSR $99D1`（動畫處理） |
| 2 | `$86E5` | `JSR $9A0D`（百位數字遞減動畫） |
| 3 | `$86ED` | `JSR $9A1F`（十位數字遞減動畫） |
| 4 | `$86F5` | 4 幀循環：載 `$87B3` 表 → `$0631` → `$9A71` → 等 4 幀 |
| 5 | `$8712` | 4 幀循環（遞增）：載 `$87B4` 表 → 同上 |
| 6 | `$8733` | 複製 `$88D2` 表到 `$0468` |
| 7 | `$879E` | **腳本跳轉**: 回存 `$58/$59/$5A` → `$4D/$4E/$56` → `JMP $84D7` |

### 腳本系統架構

```
scene_id → $8AEE 表查找 → bank + offset
  → $8464: 構造 $A0xx 指標 (reg7 區域)
  → $C4B9: 切換 bank
  → 讀 2-byte 指針 → $4D/$4E
  → $84E3: 主解釋器循環
     ↓
  $84E7: LDA ($4D),Y  ← 讀取 opcode
     ↓
  $84EB: CMP #$D8 → 分三區:
    $00-$D7: 文字輸出 ($88CA)
    $D8-$DF: 控制碼 ($8AE6 表)
    $E0-$E7: 光標控制
    $E8-$FF: 跳轉表分派
```

### 腳本跳轉 save/restore 機制

```
save ($877A):
  $58 = $4D + 3   ; 保存返回位址 (跳過嵌套調用指令)
  $59 = $4E       ; 
  $5A = $56       ; 保存 bank 號

restore ($879E):
  $4D = $58       ; 恢復指針
  $4E = $59       ;
  $56 = $5A       ; 恢復 bank
  JMP $84D7       ; 繼續執行
```

### Bank 3 玩家資料結構

Bank 3 (`$8000-$BFFF`) 為雙重用途：
- **`$8000`-`$9FFF`** (reg6): 程式碼/指標表區域
- **`$A000`-`$BFFF`** (reg7): 玩家記錄資料

#### 指標表 (`$8000`)

Bank 3 開頭為 16 個 **2-byte LE 指標**，指向同 bank 的 `$A0xx` 區域：

| 索引 | 指標 | ROM offset | 記錄大小 |
|---|---|---|---|
| 0 | `$A020` | `0x8030` | 641 bytes |
| 1 | `$A2A1` | `0x82B1` | 901 bytes |
| 2 | `$A626` | `0x8636` | 142 bytes |
| 3 | `$A6B4` | `0x86C4` | 40 bytes |
| 4 | `$A6DC` | `0x86EC` | 652 bytes |
| ... | ... | ... | ... |
| 15 | `$B731` | `0x9741` | 最後一筆 |

#### 記錄內容範例

**Player 0** (`$A020`, ROM `0x8030`, 641 bytes):
```
E8 01 F9 24 FB 01 34 DE ED 69 E9 ED 53 DD EF E1
40 3E 65 7D 4D 06 27 00 36 18 2E 3F 3F 3F FC DC
12 AF 0B 7A DB 12 02 16 00 0A 19 14 07 A0 07 10
```

觀察到的重複模式:
- `$12 $AF $0B` — 出現在多個玩家記錄中
- `$3F $3F $3F` — 填充/分隔標記
- `$FC` — 記錄內部段分隔
- `$E8`, `$ED`, `$F9`, `$FB` — （此處為資料標記，非 opcode）

#### 同一值出現在多個 bank

Bank 3-6 中掃描到 200-1000 範圍的 16-bit 值分布：
- Bank 3: 383 處
- Bank 4: 375 處
- Bank 5: 356 處
- Bank 6: 146 處

**注意**: 這些 $E8-$FC byte 在腳本中作 opcode 使用，但在玩家資料區中可能用作屬性標記（Type-Length-Value 編碼），兩者語義不同。

### 場景表 (`$8AEC`)

`$8464` 函數按 scene ID 查表分配 bank 和 offset：

| 場景 ID 範圍 | Bank | $8000 指標數 | 說明 |
|---|---|---|---|
| `$00`-`$0F` | 3 | 16 | 隊伍 1 或前期場景 |
| `$10`-`$1F` | 4 | 15 | 隊伍 2 或中期場景 |
| `$20`-`$5F` | 5 | 15 | 隊伍 3 或後期場景 |
| `$60`-`$FF` | 6 | - | 雜項 + 數字元數據 |

各 bank 的 `$8000` 指標表 `<indirect ref>` 表示由程式碼間接存取（非直接 ROM 定址）。

### Bank 3 全域 Marker 統計（前 5 筆記錄）

| Marker | 次數 | 推測用途 |
|---|---|---|
| `$FC` | 103 | 記錄段終止/分隔符 |
| `$FB` | 57 | flag，值多為 `$01`/`$02` |
| `$ED` | 44 | 重要屬性（值範圍: 1-105, 常見 69/83/53/49） |
| `$F7` | 37 | 後續控制（常接 `$F3`） |
| `$F3` | 36 | 數位顯示控制（`$00`=緩衝, `$FF`=雙位, others=單位） |
| `$FA` | 23 | 與 `$F9` 成對，控制旗標 |
| `$F5` | 20 | 場景狀態（值 `$04`, `$FF` 等） |
| `$EF` | 15 | 座標/位置資料 |
| `$F2` | 15 | 行數控制 |
| `$F9` | 14 | 旗標清除（`AND #$FB`） |
| `$EA` | 14 | 動畫相關 |
| `$F8` | 11 | 讀 2 bytes → `$EC`/`$ED` RAM |
| `$E8` | 8 | 記錄類型標記（值 `$01`） |
| `$EC` | 5 | 音效（在資料區中語義不同於 opcode） |

### `$8AF7` 場景資料讀取器

從 `($63)` 指針讀取場景記錄的結構化資料：
- `+0,+1`: 指針（讀入 `$63/$64`）
- `+2`: → `$75`
- `+3`: → `$76`
- `+4`: → `$48` (AND `#$3F`, → 數位值，可能是百位)
- `+4`: → 同時 rotated into `$5B`
- `+5`: → `$5E`
- `+6`: → `$5F`
- `+7`: → `$5C` (AND `#$F8`, → palette/base)

每條記錄至少 8 bytes 結構化欄位。
